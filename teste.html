📋 Lista de Verificação Manual

Testes a realizar manualmente:

  • ✅ Busca real de dados: Teste com cidades reais como "São Paulo, SP" ou "New York, NY"
  • 🌍 API de Geocodificação: Verifique se a localização é convertida em coordenadas
  • 📊 Dados de qualidade do ar: Verifique se IQA, poluentes e recomendações são exibidos
  • ⚠️ Tratamento de erros: Teste com localizações inexistentes
  • 🔄 Indicador de loading: Observe o spinner durante a busca
  • 🎨 Códigos de cores: Verifique se IQA usa cores corretas (verde=bom, vermelho=ruim)
  • 💡 Recomendações de saúde: Veja se são adequadas para cada nível de IQA
  • 📱 Responsividade: Teste em diferentes tamanhos de tela
  • 🌐 APIs públicas: Confirme uso do Nominatim e Open-Meteo

🎯 Critérios de Aceitação

A aplicação deve:

  1. ✅ Exibir um formulário com campos de cidade, estado e país
  2. ✅ Marcar cidade e estado como campos obrigatórios
  3. ✅ Validar campos obrigatórios antes do envio
  4. ✅ Mostrar mensagens de erro claras
  5. ✅ Assumir EUA como país padrão quando não informado
  6. ✅ Buscar coordenadas geográficas usando API Nominatim
  7. ✅ Obter dados de qualidade do ar da API Open-Meteo
  8. ✅ Exibir IQA atual com código de cores apropriado
  9. ✅ Mostrar poluente dominante e concentrações
  10. ✅ Fornecer recomendações de saúde baseadas no IQA
  11. ✅ Tratar erros de API de forma elegante
  12. ✅ Exibir indicador de loading durante busca
  13. ✅ Ter design responsivo e moderno
  14. ✅ Organizar código de forma maintível
